ValiDatas — Guide opérateur

Note

Ce guide s’adresse aux personnes qui utilisent la suite ValiDatas pour nettoyer les concepts et évaluer les expertises. Aucune intervention technique n’est requise.

Ordre recommandé : cleanKeywords d’abord, validExpertises ensuite.

Avant de commencer

Les deux outils partagent la même configuration. Vérifiez que vous disposez de :

Élément Description
URL de l’API Oméka S Ex. : https://omk.univ-paris8.fr/api/
Mail du propriétaire L’adresse mail associée à la clé API
key_identity Premier jeton de la clé API
key_credential Second jeton de la clé API
ID évaluateur Votre identifiant Oméka S (pour validExpertises)
Astuce

La configuration est sauvegardée séparément dans chaque outil. Si vous utilisez les deux régulièrement, pensez à renseigner les paramètres dans chacun lors de votre première session.

🧹 cleanKeywords

Objectif

cleanKeywords détecte les concepts en doublon dans Oméka S et vous permet de les fusionner. La fusion transfère automatiquement les liens vers les EC et les évaluations existantes vers le concept conservé, puis supprime l’autre.

Configuration initiale

  1. Ouvrir cleanKeywords dans le navigateur.
  2. Cliquer sur ⚙ Paramètres.
  3. Renseigner l’URL de l’API, le mail, key_identity, key_credential et l’ID évaluateur.
  4. Dans le champ Propriétés mots-clefs des personnes, saisir les propriétés utilisées (ex. skos:hasTopConcept,dcterms:subject).
  5. Cliquer sur Enregistrer.

Scanner les doublons

Scan lettre par lettre

  1. Cliquer sur une lettre dans l’interface pour analyser tous les concepts commençant par cette lettre.
  2. Les paires suspectes détectées s’affichent avec leur type et un score de similarité.
  3. Trier les résultats par Score, Taille ou A–Z selon vos besoins.

Scan complet

Le bouton ⚡ Tout scanner analyse l’ensemble du référentiel d’un coup. Les résultats sont mis en cache (date visible à l’écran) et restent disponibles jusqu’au prochain clic sur 🔄 Actualiser.

Avertissement

Le scan complet peut être long sur un référentiel volumineux. Préférez le scan lettre par lettre pour une vérification ciblée.

Types de doublons détectés

Type Description
Wikidata Concepts pointant vers la même entité Wikidata
Textuel Libellés identiques ou quasi-identiques
Variante ortho. Même terme avec des graphies différentes
Singulier / pluriel Ex. : “réseau” et “réseaux”
Termes inclus Un terme contenu dans un autre plus long

Utilisez les filtres en haut de liste pour n’afficher qu’un type à la fois.

Traiter une paire suspecte

Pour chaque paire affichée :

  • Rejeter → faux positif, les deux concepts sont légitimement distincts, la paire disparaît de la liste
  • Fusionner → → ouvre la fenêtre de confirmation

Fenêtre de fusion

La fenêtre affiche :

  • le concept Source (sera supprimé)
  • le concept Cible (sera conservé)
Mise en garde

Vérifiez bien quel concept est Source et lequel est Cible avant de confirmer. La fusion est définitive et irréversible.

En cas de doute sur le sens à choisir, préférer conserver le concept le plus utilisé (taille la plus grande) comme Cible.

  1. Vérifier Source → Cible.
  2. Cliquer sur Fusionner définitivement.
  3. L’opération apparaît dans le Journal des opérations en bas de page.

Bonnes pratiques

  • Travailler lettre par lettre plutôt que tout scanner d’un coup lors des premières sessions — cela permet de s’approprier le référentiel.
  • Consulter le Journal des opérations en fin de session pour vérifier qu’aucune fusion non voulue n’a eu lieu.
  • Ne pas lancer cleanKeywords et validExpertises simultanément sur le même référentiel.

✅ validExpertises

Objectif

validExpertises permet d’attribuer une valeur d’expertise (-100 à +100) à chaque concept associé à un EC, et de stocker cette évaluation dans Oméka S sous forme d’item Expertise.

Configuration initiale

  1. Ouvrir validExpertises dans le navigateur.
  2. Cliquer sur ⚙ Paramètres.
  3. Renseigner les mêmes informations API que pour cleanKeywords, ainsi que l’ID évaluateur.
  4. Cliquer sur Enregistrer.
Note

Résultat attendu : votre nom apparaît dans l’en-tête à côté de la mention Expert. Sinon, vérifier l’URL de l’API et l’ID évaluateur.

Démarrer une évaluation

Par laboratoire

  1. Sélectionner un laboratoire dans le menu déroulant Enseignants-Chercheurs.
  2. Naviguer entre les EC avec ← Précédent / Suivant →.
  3. Cliquer sur un EC pour charger sa fiche.

Par recherche directe

Taper quelques lettres dans le champ Recherche… et sélectionner la personne dans l’autocomplétion.

Astuce

Un lien avec ?idAuthor=XXX dans l’URL charge la personne automatiquement à l’ouverture.

Évaluer les expertises

Une fois une personne chargée, ses concepts s’affichent sous forme de cartes.

  1. Déplacer le slider entre -100 et +100.
  2. Cliquer sur Ajouter (nouveau) ou Modifier (existant) pour enregistrer.
Avertissement

Déplacer le slider ne suffit pas — il faut cliquer sur Ajouter ou Modifier pour sauvegarder dans Oméka S.

Échelle recommandée

Plage Signification
-100 à -30 Expertise faible ou non pertinente
-29 à +29 Neutre / incertain
+30 à +100 Expertise confirmée

Ajouter un mot-clé manquant

  1. Utiliser le champ Ajouter un mot-clé… avec autocomplétion.
  2. Vérifier que la carte apparaît avant de saisir une valeur.

Filtrer et trier

Filtre Effet
Sans votre expertise Concepts non encore évalués
Avec votre expertise Concepts déjà évalués
Positifs / Négatifs Filtrer par polarité

Les colonnes Titre et Rang sont triables. Les concepts avec un score < 5 sont masqués par défaut — Tout afficher les révèle.

Bonnes pratiques

  • Utiliser le filtre Sans votre expertise pour avancer méthodiquement.
  • Recharger la personne après une série d’évaluations pour vérifier la persistance.
  • Utiliser une échelle cohérente d’un EC à l’autre.
