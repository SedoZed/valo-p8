ValiDatas — Documentation technique

Note

Cette documentation s’adresse aux personnes chargées d’installer, configurer, maintenir ou faire évoluer les outils de la suite ValiDatas.

Dépôts GitHub : samszo/cleanKeywords · samszo/validExpertises

Vue d’ensemble technique

Les deux outils sont des applications web front-end uniquement (HTML + CSS + JavaScript vanilla), sans back-end propre. Toutes les données transitent via l’API REST d’Oméka S. Ils partagent la même structure de configuration et les mêmes conventions de code.

Prérequis spécifiques

En plus des prérequis communs (voir section suivante), cleanKeywords requiert :

  • Des droits de suppression sur les items Oméka S (la fusion supprime le concept Source).
  • La propriété valo:responsableDe pour identifier les laboratoires de l’évaluateur.
  • Les concepts accessibles via les propriétés listées dans personPropExp.

Paramètres de configuration

Paramètre Description
apiOmk URL de base de l’API Oméka S
mail Mail du propriétaire de la clé API
ident key_identity
key key_credential

personPropExp Propriétés mots-clefs des personnes, séparées par des virgules

Flux API — détection de doublons

Scan par lettre

GET /items?item_type[]=<type_concept>
          &property[0][property]=dcterms:title
          &property[0][type]=sw
          &property[0][text]=<lettre>

Comparaison des paires

La détection se fait côté client : les libellés sont comparés par algorithme de similarité textuelle, recoupés avec les identifiants Wikidata des concepts (skos:exactMatch ou équivalent) pour identifier les doublons sémantiques.

Flux API — fusion

La fusion s’opère en plusieurs étapes :

  1. Récupérer les items liés au concept Source (EC, Expertises) via leurs propriétés de relation.
  2. Mettre à jour chaque item lié pour remplacer la référence Source par la référence Cible — PUT /items/{id}.
  3. Supprimer le concept Source — DELETE /items/{id_source}.
Mise en garde

La suppression est définitive. Il n’existe pas de mécanisme d’annulation côté application. En cas d’erreur, la restauration passe par un accès direct à Oméka S (interface admin ou API).

Journal des opérations

Chaque fusion est tracée dans le journal affiché en bas de page. Ce journal est en mémoire uniquement (non persisté) — il est perdu au rechargement de la page.

Avertissement

Si la traçabilité des opérations est critique, envisager d’ajouter une persistance du journal (localStorage ou export CSV) comme évolution future.

Installation

# Cloner les deux dépôts
git clone https://github.com/samszo/cleanKeywords.git
git clone https://github.com/samszo/validExpertises.git

# Servir depuis un serveur web local
cd cleanKeywords && python3 -m http.server 8080
cd validExpertises && python3 -m http.server 8081
Avertissement

CORS : si Oméka S est sur un domaine différent, configurer les en-têtes CORS sur le serveur Oméka S pour les deux origines.

Sécurité

  • Ne jamais committer key_identity / key_credential dans authParams.js si les dépôts sont publics.
  • Créer des clés API avec les privilèges minimaux nécessaires à chaque outil.
  • cleanKeywords requiert des droits de suppression — limiter son accès aux opérateurs de confiance.

Points d’attention pour la maintenance

Requêtes synchrones (XMLHttpRequest)

Avertissement

Les deux outils utilisent des XMLHttpRequest synchrones dans certaines parties de omk.js. Ce comportement est déprécié. Une migration vers fetch() avec async/await est recommandée comme priorité d’évolution.

Cohérence inter-outils

Après une session de fusion dans cleanKeywords, les items Expertise dans validExpertises qui référençaient le concept Source sont automatiquement mis à jour (la fusion le garantit). Vérifier toutefois après une fusion massive que les évaluations existantes sont bien rattachées au bon concept.

Mise à jour de D3

D3 est chargé localement dans les deux outils (assets/js/d3.min.js). Mettre à jour les deux fichiers simultanément pour éviter des divergences de comportement.

Dépannage

Symptôme Cause probable Action
Évaluateur absent de l’en-tête creatorId incorrect Vérifier l’ID dans les paramètres
Aucun concept affiché personPropExp mal configuré Vérifier les propriétés
