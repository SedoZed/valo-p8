Formation Omeka S pour non-informaticiens

Guide de prise en main basique de l’interface d’administration

Auteur·rice

Cédric BOUCLY

Date de publication

13 octobre 2027

Mots clés

Omeka S, base de données, métadonnées, ontologies, initiation


Préambule

Objectifs : À la fin de la formation, les participants seront capables de :

  1. Comprendre la logique interne d’Omeka S (Items, Item Sets, Relations, etc)
  2. Naviguer dans la base de données sans confusion entre les différents types d’objets
  3. Effectuer quelques modifications spécifiques directement dans l’interface Omeka S.


1. Le paysage global

AstuceObjectif

Comprendre la logique interne d’Omeka S.

Concepts Clés

Concept Définition Simplifiée À retenir
Base de données Là où on crée, modifie, organise le contenu Interface principale de travail
Site/CMS Outil de déploiement & configuration de site interne à Omeka S Non utilisé pour le moment
API Mécanisme d’exposition programmatique des données Export/Connexion vers des outils externes

Pourquoi Omeka S ?

Omeka S est conçu pour structurer des collections patrimoniales (musées, archives, bibliothèques) où les métadonnées sont aussi importantes que les fichiers eux-mêmes. L’accent est mis sur la qualité et l’interopérabilité des données : chaque item est décrit selon des vocabulaires standards, ce qui rend les informations fiables, cohérentes et réutilisables.

2. L’Item, Unité de Base

AstuceObjectif

Comprendre qu’un item est l’entrée fondamentale dans la base.

Anatomie d’un Item

Un item = une entrée dans la base de données. Il contient trois parties :

Élément Description
o:id (o:id) Numéro unique
Métadonnées Champs descriptifs
Médias Fichiers attachés (pas vraiment utile ici)



EXEMPLE :

Élément Valeur d’exemple
o:id 1543
dcterms:title “Carte postale du viaduc de Caen, 1995”
dcterms:creator Archives départementales
dcterms:date 2014-06-12
Média(s) carte_postale_caen.jpg (2,4 Mo)

L’identifiant “o:id”

Définition : Numéro unique attribué automatiquement à la création

Utilité : Ne change jamais, même si le titre ou les métadonnées changent

À retenir : “C’est le numéro de matricule de l’item”

Trouver un Item

Avant de modifier un item, il faut d’abord le repérer.

Méthodes de Recherche

Méthode Comment faire Quand l’utiliser
Barre de recherche En haut de la page Items Mot-clé, un titre, ou une partie du contenu
Filtres par Item Set Menu latéral → “Item Sets” Chercher dans une collection spécifique
Navigation via o:id URL directe (/admin/show/items/[NUMÉRO]) Si l’o:id précis est déjà connu

Utiliser “Advanced Search”

Le module Advanced Search enrichit la recherche de base avec des filtres avancés. Celui-ci permet de :

  • Recherche par plusieurs critères simultanément (ex: titre + date + créateur)
  • Tri personnalisable : par date, par titre, par classe/item set, par pertinence, etc.
  • De faire des requêtes “noSQL”

Comment l’utiliser

Action Emplacement Résultat
Rechercher Bouton “Recherche avancée” au-dessus de la liste Formulaire avec champs multiples
Filtrer Panneau latéral gauche Affiner par Item Set, vocabulaire, etc.
Trier Menu déroulant “Trier par” Ordonner les résultats

3. Vocabulaires et Propriétés

AstuceObjectif

Comprendre d’où viennent les champs lorsqu’on ouvre un item.

Principe

Les métadonnées d’un item ne sont pas inventées au hasard. Elles proviennent de vocabulaires, qui définissent :

  • Quels champs existent
  • Ce que chaque champ signifie
  • Quel type de valeur est attendu

Exemple connu : Dublin Core (dcterms:xxx) → Le vocabulaire le plus utilisé internationalement, présent par défaut dans Omeka S, et ayant des propriétés comme…

Propriété Signification Type attendu
dcterms:title Titre de l’item Texte
dcterms:creator Créateur Texte / Ressource
dcterms:date Date Texte (ISO recommandé)
dcterms:description Description Texte long
dcterms:type Type de ressource Texte / Controlled vocabulary

Deux principaux types de valeurs

Les valeurs contenues dans chacune des propriétés peuvent être de différents types, dont deux principaux :

Type Description Exemple
Texte libre Du texte classique, écrit directement Titre, description, dates (parfois), …
Ressource Pointe vers un autre item “dcterms:creator” pointant vers l’item “Jean Dupont” (o:id 86), …
AstuceOù consulter le vocabulaire ?

Les vocabulaires gérés par l’instance Omeka S sont consultables dans l’onglet “Vocabularies” de l’interface (utile pour se renseigner). Les propriétés apparaissent directement lors de la création ou de l’édition d’un item.

4. Organisation et Relations

AstuceObjectif

Comprendre comment les items s’organisent et se lient entre eux.

Partie A : Les Item Sets

Définition

Un Item Set est un regroupement logique d’items (collection thématique, période, etc.)

Analogie

  • Item → Item Set
  • Un livre → Une étagère
  • Un objet → Une vitrine muséale

Règles Importantes

  • Un item peut appartenir à plusieurs Item Sets
  • Un Item Set ne contient pas de médias directs (seulement des références à des items)

Partie B : Relations Entre Items via o:id

Omeka S n’est pas une simple liste de fiches indépendantes, mais plutôt un réseau de données reliées entre elles grâce aux propriétés ayant un contenu de type “ressource”, c’est-à-dire une redirection vers un autre item — comme un lien hypertexte plutôt que de recopier un titre.

Exemple

Item “Document historique” (o:id: 142) └── dcterms:creator → Item “Personne” (o:id: 89) La relation est faite via le o:id de l’item cible.

Pourquoi pas juste du texte ? | Relation via o:id | Texte Libre | | —————– | ———– | | Données structurées et interrogables | Impossible de faire des requêtes | | Liens exploitables par l’API | Informations limitées | | Mise à jour centralisée | Duplication des données |

Retenir : Les deux approches ont leur place. Mais dès qu’une donnée peut pointer vers autre chose, faites-le : vos données seront plus riches et cohérentes.

5. Le module “ScanR”

AstuceObjectif

Comprendre comment importer des données externes (personnes) via l’API ScanR.

Qu’est-ce que ScanR ?

ScanR est la plateforme du Ministère de l’Enseignement Supérieur et de la Recherche qui recense :

  • Chercheurs et leurs affiliations
  • Domaines de recherche
  • Publications et distinctions

Le module ScanR pour Omeka S, développé par Samuel Szoniecky, permet d’importer directement sur Omeka S les fiches personnes depuis cette base.

Effectuer un import :

Il existe deux façons d’importer les données depuis la base ScanR :


Import individuel
Étape Action Résultat
1 Accéder à l’onglet “ScanR” dans l’interface Formulaire de recherche
2 Rechercher une personne (nom, ORCID, structure) Résultats depuis ScanR
3 Sélectionner et valider l’import Création ou modification de l’item “Personne” avec les métadonnées de ScanR
4 Vérifier/mettre à jour les liens (ex: dcterms:creator → structure) Données cohérentes


Import massif
Étape Action Résultat
1 Accéder à la liste souhaitée d’items, et sélectionner ces derniers —
2 Cliquer sur “Batch actions”, puis “Edit selected”, puis sur le bouton “Go” Une nouvelle fenêtre s’ouvre
3 Cocher la case “Merge item data with Scanr first response”, puis “Submit” Retour à la liste des items, l’import se fait en arrière-plan
4 Vérifier/mettre à jour les liens (ex: dcterms:creator → structure) Données cohérentes

Limites à connaître

  • Noms complexes : Certaines orthographes ou noms composés peuvent poser problème à l’extraction
  • Biais de couverture : Les concepts sont basés sur les publications du MESR → certaines communautés peuvent être moins représentées
  • Résultats consultables limités : Un bug empêche d’afficher les EC trouvés au-delà de trois noms (import indiviuel uniquement)
  • Configuration préalable : Nécessite un template et un item set dédiés aux personnes importées

6. Atelier Pratique

Consignes : Accéder à l’item qui vous désigne, et éditez-le pour ajouter des informations complémentaires. Ces données doivent contenir au moins une fois du texte et une fois une ressource. Assurez-vous aussi que l’item que vous éditez est bien dans un Item Set cohérent (le rajouter ou le modifier si besoin).

  • Exemple d’éléments à inclure : description, date de naissance, un/des laboratoire(s), …

Critère de réussite : Les nouvelles informations apparaissent sur la fiche, et l’élément “ressource” redirige bien vers un autre item.

AstuceUne fois l’exercice validé…

Envie de vous exercer sur autre chose ? N’hésitez pas, tant que le formateur se trouve pas loin pour vous aider ! :)

En résumé

Ce que tu veux faire Où aller
Modifier le titre/description d’un item L’item lui-même (édition)
Regrouper plusieurs items Créer un Item Set
Relier un item à un autre Utiliser une propriété de type “ressource”
Ajouter un fichier Section “Médias” de l’item
Chercher un item Barre de recherche ou navigation par ID
AvertissementQuand appeler l’administrateur
  • Besoin de créer un nouveau vocabulaire
  • Problème de connexion ou droits d’accès
  • Erreur système après sauvegarde
  • Besoin de modifications hors du périmètre autorisé

Annexes

Glossaire rapide

Terme Signification
API Mécanisme technique d’exposition des données vers d’autres systèmes (hors scope)
Base de données Stockage central où sont créées, modifiées et organisées toutes les données
Module Extension qui ajoute des fonctionnalités à Omeka S (ex: ScanR)
Item Entrée dans la base (photo, document, objet…)
Item Set Regroupement logique d’items (collection, fonds thématique)
Média Fichier attaché à un item (peu évoqué dans notre cas)
o:id Identifiant unique permanent attribué automatiquement
Propriété Champ spécifique (ex: dcterms:title)
Relation Lien actif entre items via une propriété de type “ressource”
Ressource Propriété pointant vers un autre item (vs. texte libre)
Vocabulaire Ensemble de propriétés et de classes définissant les champs possibles
