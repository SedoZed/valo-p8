validExpertises — Guide opérateur

Note

Ce guide s’adresse aux personnes qui utilisent l’application pour évaluer des expertises au quotidien. Aucune intervention technique sur le code n’est nécessaire.

Version courte imprimable disponible sur le dépôt GitHub : FICHE_OPERATEUR_1_PAGE.md.

Qu’est-ce que validExpertises ?

validExpertises est une application web qui permet d’évaluer les expertises thématiques des Enseignants-Chercheurs (EC) rattachés au Laboratoire Paragraphe (Paris 8).

Concrètement, pour chaque EC chargé dans l’application, vous visualisez ses mots-clés (concepts scientifiques extraits d’Oméka S), et vous pouvez attribuer à chacun une valeur d’expertise allant de -100 (non pertinent) à +100 (fortement pertinent). Ces évaluations sont enregistrées directement dans Oméka S via l’API.

L’outil fait partie de l’écosystème CartoExpert ; il est accessible à l’adresse : 👉 https://valorisation.humanum-p8.fr/outils/validExpertises/

Avant de commencer

Avant d’ouvrir l’application, vérifiez que vous disposez des informations suivantes :

Élément Description
URL de l’API Oméka S Ex. : https://omk.univ-paris8.fr/api/
Mail du propriétaire L’adresse mail associée à la clé API
key_identity Premier jeton de la clé API Oméka S
key_credential Second jeton de la clé API Oméka S
ID évaluateur L’identifiant Oméka S de votre fiche personnelle
Astuce

Ces informations sont fournies par l’administrateur technique du projet. Si vous ne les avez pas, contactez la Direction de la Recherche de Paris 8.

Configuration initiale (une seule fois)

La configuration n’est à réaliser qu’une seule fois par navigateur. Elle est ensuite conservée automatiquement.

  1. Ouvrir l’application dans votre navigateur.
  2. Cliquer sur le bouton ⚙ Paramètres en haut de page.
  3. Renseigner les champs de la fenêtre :
    • URL de base de l’API → coller l’URL fournie
    • Mail de l’utilisateur Oméka S → votre mail
    • key_identity et key_credential → les deux jetons de la clé
    • ID évaluateur → votre identifiant Oméka S
    • Laisser les autres champs à leur valeur par défaut sauf indication contraire
  4. Cliquer sur Enregistrer.
Note

Résultat attendu : votre nom (ou identifiant) apparaît dans l’en-tête de la page, à côté de la mention Expert. Si rien n’apparaît, vérifiez l’URL de l’API et l’ID évaluateur.

Démarrer une session d’évaluation

Sélectionner un EC par laboratoire

Le menu déroulant Enseignants-Chercheurs vous permet de parcourir les EC par laboratoire.

  1. Sélectionner un laboratoire dans la liste déroulante.
  2. Naviguer entre les EC avec les boutons ← Précédent et Suivant →.
  3. Cliquer sur un EC pour charger sa fiche.

Rechercher directement une personne

  1. Taper quelques lettres du nom dans le champ Recherche… (autocomplétion).
  2. Sélectionner la bonne personne dans la liste proposée.
  3. Vérifier que le bandeau affiche bien le nom et l’identifiant de la personne chargée.
Astuce

Raccourci possible : si quelqu’un vous partage un lien avec un paramètre ?idAuthor=XXX dans l’URL, la personne est chargée automatiquement à l’ouverture — pas besoin de rechercher.

Évaluer les expertises

Une fois une personne chargée, ses mots-clés s’affichent sous forme de cartes.

Lire une carte

Chaque carte affiche :

  • le titre du mot-clé (concept scientifique)
  • la propriété source (d’où vient ce mot-clé dans Oméka S)
  • un slider de -100 à +100
  • votre évaluation existante, si elle a déjà été saisie

Saisir ou modifier une valeur

  1. Déplacer le slider vers la valeur souhaitée.
  2. Selon la situation :
    • Ajouter → si aucune évaluation n’existe encore pour ce mot-clé
    • Modifier → si vous souhaitez corriger une valeur existante
    • Supprimer → pour retirer complètement votre évaluation
Avertissement

Cliquer sur Ajouter ou Modifier est indispensable pour enregistrer la valeur dans Oméka S. Déplacer le slider seul ne suffit pas.

Échelle de valeurs recommandée

Plage Signification
-100 à -30 Expertise faible ou non pertinente
-29 à +29 Neutre / incertain
+30 à +100 Expertise confirmée

Ajouter un mot-clé avant évaluation

Si un concept pertinent n’est pas encore présent dans la liste de la personne :

  1. Utiliser le champ Ajouter un mot-clé… (avec autocomplétion).
  2. Taper quelques lettres et sélectionner le concept dans la liste.
  3. Vérifier que la carte apparaît dans la grille avant de saisir une valeur.
Mise en garde

Le mot-clé ajouté doit exister dans Oméka S pour apparaître dans l’autocomplétion. Si vous ne le trouvez pas, signalez-le à l’administrateur pour qu’il soit créé au préalable.

Filtrer et trier les mots-clés

Des outils sont disponibles pour travailler de manière systématique sur de longues listes.

Filtres disponibles

Filtre Effet
Tous Affiche tous les mots-clés
Sans votre expertise Mots-clés que vous n’avez pas encore évalués
Avec votre expertise Mots-clés que vous avez déjà évalués
Positifs Uniquement les valeurs > 0
Négatifs Uniquement les valeurs < 0

Tri

Les colonnes Titre et Rang sont triables par clic sur leur en-tête (↕).

Mots-clés peu pertinents

Par défaut, les mots-clés dont le score est inférieur à 5 et qui figurent au-delà des 20 premiers sont masqués automatiquement. Le bouton Tout afficher permet de les révéler.

Bonnes pratiques opérateur

  • Utiliser une échelle cohérente d’une personne à l’autre pour que les comparaisons soient exploitables.
  • Préférer le filtre Sans votre expertise pour avancer méthodiquement sans oubli.
  • En cas de doute sur une valeur, noter l’ID Oméka S de la personne et reprendre plus tard.
  • Toujours recharger la personne après une série d’évaluations pour vérifier la persistance des données.

Résolution de problèmes fréquents

Erreur HTTP au chargement

Causes possibles : URL de l’API incorrecte, clés API expirées, instance Oméka S inaccessible.

Action : Vérifier les paramètres (⚙), relancer depuis zéro si nécessaire.

Aucun mot-clé ne s’affiche

Causes possibles : La personne ne possède pas de concepts dans la propriété configurée, ou le type de la personne n’est pas autorisé.

Action : Vérifier avec l’administrateur que la fiche Oméka S de la personne est correctement renseignée.

Impossible d’enregistrer (bouton Ajouter sans effet)

Causes possibles : Droits insuffisants sur la clé API, propriétés cibles inexistantes dans Oméka S.

Action : Contacter l’administrateur pour vérifier les droits de la clé API.

Le bon nom n’apparaît pas dans l’autocomplétion

Action : Essayer une autre partie du nom, ou utiliser directement le paramètre URL ?idAuthor=ID_OMEKA.

Procédure de reprise après interruption

En cas de doute sur l’état de sauvegarde :

  1. Notez l’ID Oméka S de la personne en cours.
  2. Rechargez la page.
  3. Rouvrez ⚙ Paramètres et vérifiez les valeurs.
  4. Rechargez la personne via l’autocomplétion ou ?idAuthor=ID.
  5. Vérifiez les valeurs affichées avant toute nouvelle modification.
